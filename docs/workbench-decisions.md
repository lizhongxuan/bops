# 工作台改造默认决策

> 说明：此文件用于记录当前实现的默认范围与技术选型。若与实际需求不符，可随时调整并同步到设计与实现。

## 1. 产品范围（默认）
- 首页 `/` 替换为工作台视图。
- 保留旧首页视图为 `/home`（便于回退或对比）。
- 工作台功能包含：拖拽节点、连线、AI 生成/重生成、模板库、沙盒校验与自动修复、运行执行反馈、右侧聊天抽屉折叠。
- 运行模式：以“线性流程优先，弱分支支持”为默认策略，复杂 DAG 作为后续扩展。

## 2. 图编辑技术选型（默认）
- **前端图编辑器优先选型**：`@vue-flow/core`（与 Vue 生态适配、内置拖拽/连线/缩放）。
- **Phase 1**：基于 Vue Flow 的基础能力（拖拽/连线/选择/删除 + 简单布局）。
- **Phase 2**：根据业务复杂度补充自定义节点渲染与更强的布局算法。

## 3. 性能与容量目标（默认）
- 画布节点数上限：**200**
- 边数量上限：**400**
- 单次运行事件吞吐：**<= 50 events/sec**
- 日志长度上限：**10,000 行**
- 自动布局耗时：**< 300ms**（200 节点）

## 4. 兼容性与渐进策略
- YAML 仍是执行真值源；Graph 为编辑与交互层缓存。
- Graph 与 YAML 的一致性由同步规则与冲突检测保障。
